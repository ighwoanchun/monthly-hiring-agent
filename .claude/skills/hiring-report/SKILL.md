# hiring-report

월간 채용 분석 리포트를 생성합니다.

## 실행 방법

1. `data/` 디렉토리의 엑셀 파일(지표분석용_sheet.xlsx)을 읽는다
2. `scripts/analyze.py` 헬퍼 함수를 활용하여 데이터를 가공한다
3. 아래 리포트 구조에 맞춰 마크다운 리포트를 생성한다
4. `output/` 디렉토리에 `{년도}년_{월}월_월간_채용_분석_리포트.md`로 저장한다
5. Confluence 업로드 및 Slack 알림 (MCP 연동 시)

## 데이터 구조

| 시트명 | 기준 | 용도 |

| 월통합분석 | 월별 집계 | KPI 추이, 매출 |

### 수수료단가별합격자수raw 상세

컬럼: `hire_month`(합격월) / `job_category`(**영문** 표기) / `fee_band` / `hired_count`

- **fee_band는 표기와 달리 배타적 구간**이다 (구간별 합 = 해당 그룹 총 합격수). "이상" 표기를 누적으로 오해하지 말 것.
  - `2백만원미만합격자` = ~2백 / `2백만원이상합격자` = 2백대(2~3백) / `3백만원이상합격자` = 3백대 / `4백만원이상합격자` = 4백대 / `5백만원이상합격자` = 5백+
- **모집단이 다른 raw 탭과 다름**: 이 탭은 "수수료가 확정(청구)된 합격자"만 포함하는 서브셋이다. `합격기준리드타임_raw`의 hire_count 합계와 일치하지 않는 것이 정상(커버리지 대략 50~65%, 직군별로 유사). **"직군별 합계 = hire_cnt 일치 필수" 규칙을 이 탭에는 적용하지 않는다.** 리포트에는 반드시 "수수료 확정 합격자 기준(전체 합격자의 약 OO%)" 각주를 단다.
- **job_category 영→한 매핑** (다른 raw 탭과 결합 시 사용):

## 리포트 구조

### Part A. 실적 분석 (합격기준)
1. Executive Summary - 6개 지표 + 테이블형태
2. 월별 KPI 추이 - 4개월 + 열에 전월대비 증감
3. 매출 구조 - 수수료/정액제/광고 + 테이블형태
4. 건당 수수료 단가 추이 - 수수료매출/합격수/건당단가 + 테이블형태
   - 4-1. 수수료 단가 구성 분석 (직군별) - 수수료단가별합격자수raw 활용
     - 전사 구간별(~2백/2백대/3백대/4백대/5백+) 분포 추이, 4개월 + 구간별 비중 + 가중평균단가(백만원)
     - 직군별 x 구간별 매트릭스 (TOP7 직군 + 기타, 당월)
     - 직군별 가중평균단가 MoM + 상태(상승/하락/보합)
     - 종합 인사이트: 단가 변동을 주도한 직군 → 방향성 → 시사점/액션 3문장
     - 각주: "수수료 확정 합격자 기준(전체 합격자의 약 OO%)"
5. 합격자 분석
   - 5-1. 직군별/합격수/비중/전월/전월대비증감%/상태/리드타임
   - 5-2. 기업규모별
6. 성과 평가

### Part B. 파이프라인 분석 (지원기준)
7. 지원 현황 - 직군별/기업규모별
8. 퍼널 전환 분석 (핵심)
9. 익월 예측

### 공통
10. 리스크 & 기회
11. 액션 아이템

## 분석 규칙

- MoM = (당월-전월)/전월 x 100
- 상태: >+5% → 상승, <-5% → 하락, 그 외 → 보합
- 리드타임 가중평균 = sum(리드타임 x 합격수) / sum(합격수)
- **직군별/기업규모별 합계 = hire_cnt 일치 필수** (단, 수수료단가별합격자수raw는 예외 - 위 상세 참조)
- TOP 10 외 → "기타", null → "미분류" (수수료 단가 구성 분석은 TOP 7 외 → "기타")
- 수수료 구간 가중평균단가(백만원) = sum(구간 중앙값 x 구간별 합격수) / sum(합격수)
  - 구간 중앙값: ~2백=1.5 / 2백대=2.5 / 3백대=3.5 / 4백대=4.5 / 5백+=5.5
